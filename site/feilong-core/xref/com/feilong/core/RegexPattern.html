147  * <h3>全部符号解释</h3>
148  * 
149  * <blockquote>
150  * <table border="1" cellspacing="0" cellpadding="4" summary="">
151  * <tr style="background-color:#ccccff">
152  * <th align="left">字段</th>
153  * <th align="left">说明</th>
154  * </tr>
155  * 
156  * <tr valign="top">
157  * <td>{n}</td>
158  * <td>n 是一个非负整数.匹配确定的 n 次.例如,'o{2}' 不能匹配 "Bob" 中的 'o',但是能匹配 "food" 中的两个 o.</td>
159  * </tr>
160  * <tr valign="top" style="background-color:#eeeeff">
161  * <td>{n,}</td>
162  * <td>n 是一个非负整数.至少匹配n 次.例如,'o{2,}' 不能匹配 "Bob" 中的 'o',但能匹配 "foooood" 中的所有 o.'o{1,}' 等价于 'o+'.'o{0,}' 则等价于 'o*'.</td>
163  * </tr>
164  * <tr valign="top">
165  * <td>{n,m}</td>
166  * <td>m 和 n 均为非负整数,其中 {@code n <= m}.最少匹配 n 次且最多匹配 m 次.例如,"o{1,3}" 将匹配 "fooooood" 中的前三个 o.'o{0,1}' 等价于 'o?'.请注意在逗号和两个数之间不能有空格.</td>
167  * </tr>
168  * <tr valign="top" style="background-color:#eeeeff">
169  * <td>?</td>
170  * <td>当该字符紧跟在任何一个其他限制符 (*, +, ?, {n}, {n,}, {n,m}) 后面时,匹配模式是非贪婪的.非贪婪模式尽可能少的匹配所搜索的字符串,而默认的贪婪模式则尽可能多的匹配所搜索的字符串.例如,对于字符串 "oooo",'o+?' 将匹配单个
171  * "o",而 'o+' 将匹配所有 'o'.</td>
172  * </tr>
173  * <tr valign="top">
174  * <td>.</td>
175  * <td>匹配除 "\n" 之外的任何单个字符.要匹配包括 '\n' 在内的任何字符,请使用象 '[.\n]' 的模式.</td>
176  * </tr>
177  * <tr valign="top" style="background-color:#eeeeff">
178  * <td>(pattern)</td>
179  * <td>匹配 pattern 并获取这一匹配.所获取的匹配可以从产生的 Matches 集合得到,在VBScript 中使用 SubMatches 集合,在JScript 中则使用 $0…$9 属性.要匹配圆括号字符,请使用 '\(' 或 '\)'.</td>
180  * </tr>
181  * <tr valign="top">
182  * <td>(?:pattern)</td>
183  * <td>匹配 pattern 但不获取匹配结果,也就是说这是一个非获取匹配,不进行存储供以后使用.这在使用 "或" 字符 (|) 来组合一个模式的各个部分是很有用.例如, 'industr(?:y|ies) 就是一个比 'industry|industries'
184  * 更简略的表达式.</td>
185  * </tr>
186  * <tr valign="top" style="background-color:#eeeeff">
187  * <td>(?=pattern)</td>
188  * <td>正向预查,在任何匹配 pattern 的字符串开始处匹配查找字符串.这是一个非获取匹配,也就是说,该匹配不需要获取供以后使用.例如,'Windows (?=95|98|NT|2000)' 能匹配 "Windows 2000" 中的 "Windows" ,但不能匹配
189  * "Windows 3.1" 中的 "Windows".预查不消耗字符,也就是说,在一个匹配发生后,在最后一次匹配之后立即开始下一次匹配的搜索,而不是从包含预查的字符之后开始.</td>
190  * </tr>
191  * <tr valign="top">
192  * <td>(?!pattern)</td>
193  * <td>负向预查,在任何不匹配 pattern 的字符串开始处匹配查找字符串.这是一个非获取匹配,也就是说,该匹配不需要获取供以后使用.例如'Windows (?!95|98|NT|2000)' 能匹配 "Windows 3.1" 中的 "Windows",但不能匹配
194  * "Windows 2000" 中的 "Windows".预查不消耗字符,也就是说,在一个匹配发生后,在最后一次匹配之后立即开始下一次匹配的搜索,而不是从包含预查的字符之后开始</td>
195  * </tr>
196  * <tr valign="top" style="background-color:#eeeeff">
197  * <td>x|y</td>
198  * <td>匹配 x 或 y.例如,'z|food' 能匹配 "z" 或 "food".'(z|f)ood' 则匹配 "zood" 或 "food".</td>
199  * </tr>
200  * <tr valign="top">
201  * <td>[xyz]</td>
202  * <td>字符集合.匹配所包含的任意一个字符.例如, '[abc]' 可以匹配 "plain" 中的 'a'.</td>
203  * </tr>
204  * <tr valign="top" style="background-color:#eeeeff">
205  * <td>[^xyz]</td>
206  * <td>负值字符集合.匹配未包含的任意字符.例如, '[^abc]' 可以匹配 "plain" 中的'p'.</td>
207  * </tr>
208  * <tr valign="top">
209  * <td>[a-z]</td>
210  * <td>字符范围.匹配指定范围内的任意字符.例如,'[a-z]' 可以匹配 'a' 到 'z' 范围内的任意小写字母字符.</td>
211  * </tr>
212  * <tr valign="top" style="background-color:#eeeeff">
213  * <td>[^a-z]</td>
214  * <td>负值字符范围.匹配任何不在指定范围内的任意字符.例如,'[^a-z]' 可以匹配任何不在 'a' 到 'z' 范围内的任意字符.</td>
215  * </tr>
216  * <tr valign="top">
217  * <td>\b</td>
218  * <td>匹配一个单词边界,也就是指单词和空格间的位置.例如, 'er\b' 可以匹配"never" 中的 'er',但不能匹配 "verb" 中的 'er'.</td>
219  * </tr>
220  * <tr valign="top" style="background-color:#eeeeff">
221  * <td>\B</td>
222  * <td>匹配非单词边界.'er\B' 能匹配 "verb" 中的 'er',但不能匹配 "never" 中的 'er'.</td>
223  * </tr>
224  * <tr valign="top">
225  * <td>\cx</td>
226  * <td>匹配由 x 指明的控制字符.例如, \cM 匹配一个 Control-M 或回车符.x 的值必须为 A-Z 或 a-z 之一.否则,将 c 视为一个原义的 'c' 字符.</td>
227  * </tr>
228  * 
229  * </table>
230  * </blockquote>
231  * 
232  * <pre class="code">
233  * {@code
234  * 1.字符类是可选自符的集合,用‘[’封装,比如[Jj],[0-9],[A-Za-z]或[^0-9].这里的-表示范围(Unicode落在两个边界之间的所有字符),^表示求补(指定字符外的所有字符 
235  * 2.有许多预定以的字符类,像\d(数字)或\p&#123;Sc&#125;(Unicode货币符号),见表12-8和12-9. 
236  * 3大多数字符与它们自身匹配,像上例中的java字符. 
237  * 4.符号.匹配任何字符(可能行终止符(line terminators)除外,这依赖于标识设置(flag settings)) 
238  * 5.\用作转义符,比如\.匹配一个句点,\\匹配一个反斜杠. 
239  * 6.^和$分别匹配行头和行尾 
240  * 7.如果X和Y都是正则表达式,则XY表示"X的匹配后面跟着Y的匹配".X|Y表示"任何X或Y的匹配" 
241  * 8.可以将量词(quantifier)用到表达式中,X+ 表示X重复1次或多次,X* 表示X重复0次或多次,X? 表示X重复0次或1次 
242  * 9.默认地,一个量词总是与使总体成功匹配的最长的可能重复匹配.可以加上后缀？(称为reluctant或stingy 匹配,用以匹配最小的重复数),和+(称为possessive或贪婪匹配,用以即使在总体匹配失败的情况下也匹配最大的重复数)来更改这种属性. 
243  * }
244  * </pre>

317     /**
318      * 大陆的电话号码 <code>{@value}</code>.
319      * 
320      * <p>
321      * 支持 (3-4位)区号(option)+(6-8位)直播号码+(1-6位)分机号(option) 的组合
322      * </p>
323      * 
324      * <h3>有效的:</h3>
325      * 
326      * <blockquote>
327      * 
328      * <ul>
329      * <li>86771588</li>
330      * <li>021-86771588</li>
331      * <li>021-867715</li>
332      * <li>86771588-888</li>
333      * </ul>
334      * 
335      * </blockquote>
336      * 
337      * <h3>无效的:</h3>
338      * 
339      * <blockquote>
340      * 
341      * <pre class="code">
342      * "",
343      * "   ",
344      * "02021-86771588-888", //区号3-4位 太长了
345      * "020-86771588888", //直播号码6-8位 太长了
346      * "021-86775" //直播号码 需要 6-8位
347      * </pre>
348      * 
349      * </blockquote>
350      * 
351      * <h3>关于分机号(extension number):</h3>
352      * 
353      * <blockquote>
354      * <p>
355      * 分机号是相对总机号说的,一般公司或企事业单位为了节省外线电话,会购置一台集团电话或电话交换机,外线或叫直线电话接到交换机上,可以设一部总机,外线的电话全部接入总机,通过总机代转,<br>
356      * 一般一个电话交换机能接入外线和分出的分机是由交换机的型号决定,各分机可以设1-6位分机号,各分机之间可通过分机号任打电话,接打外线由电话交换机自动调配线路。
357      * </p>
358      * 
359      * <p>
360      * 分机号比较理想的位数为4位,第一位可以设置成部门识别号,其他的可以根据需要设置。
361      * 千万别用三位的紧急服务号码来做分机号,如果有人用直线电话误操作的话,会带来不必要的麻烦
362      * </p>
363      * </blockquote>
364      * 
365      * @see <a href="https://en.wikipedia.org/wiki/Extension_(telephone)">Extension_(telephone)</a>
366      * @see <a href="http://regexlib.com/Search.aspx?k=phone+number&c=7&m=5&ps=20">regexlib</a>
367      */
368     public static final String TELEPHONE               = "^(\\d{3,4}-)?\\d{6,8}(-\\d{1,6})?$";

370     /**
371      * 大陆的电话号码(必须要有区号) <code>{@value}</code>.
372      * 
373      * <p>
374      * 支持 (3-4位)区号(must)+(6-8位)直播号码+(1-6位)分机号(option) 的组合
375      * </p>
376      * 
377      * <h3>有效的:</h3>
378      * 
379      * <blockquote>
380      * 
381      * <ul>
382      * <li>021-86771588</li>
383      * <li>021-867715</li>
384      * <li>021-86771588-888</li>
385      * </ul>
386      * 
387      * </blockquote>
388      * 
389      * <h3>无效的:</h3>
390      * 
391      * <blockquote>
392      * 
393      * <pre class="code">
394      * "",
395      * "   ",
396      * "86771588", //没有区号
397      * "02021-86771588-888", //区号3-4位 太长了
398      * "86771588-888", //没有区号
399      * "020-86771588888", //直播号码6-8位 太长了
400      * "021-86775" //直播号码 需要 6-8位
401      * </pre>
402      * 
403      * </blockquote>
404      * 
405      * <h3>区号</h3>
406      *
